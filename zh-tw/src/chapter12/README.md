# async / await 與非同步 runtime

到目前為止，當我們的程式需要「等待」一件事情完成——例如等使用者輸入、等檔案讀完、等網路回應——程式就只能停在那裡乾等。第 8 章（多執行緒）給了我們一個解法：多開幾條執行緒，這條在等的時候，別條可以繼續做事。

但如果你要寫的是一台同時服務一萬個連線的網路伺服器，開一萬條執行緒就太貴了——每條執行緒都要佔一塊記憶體、作業系統還要花力氣在它們之間切換。問題的本質是：這些連線**絕大部分時間都在等**（等對方傳資料過來），真正在計算的時間很少。為了「等待」就綁住一整條執行緒，太浪費了。

`async`／`await` 就是為了這種「大量等待」的情況設計的。它讓單一一條執行緒在等待的空檔，自動切去處理別的工作，等到原本那件事好了再回來繼續。本章會從怎麼用開始，一路往下挖到它背後到底是怎麼運作的——我們甚至會親手寫一個迷你版的 async 系統——最後再回到實務上最常用的 Tokio 這個 runtime，把各種好用的工具補齊。

這是全書最進階的一章。如果中途覺得卡，記得 async 是出了名地難，慢慢來、多回頭看前面幾集，是很正常的。
